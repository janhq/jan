# Gonka Broker

## Integrate Gonka Broker with Jan

[Gonka Broker](https://gonkabroker.com/) is an OpenAI-compatible API for open-source models
(such as MiniMax and Kimi) running on the decentralized Gonka network. You pay in regular currency,
and your [rate is locked when you top up](https://gonkabroker.com/gonka-api-pricing/), so costs stay
predictable. No separate account is needed for each model provider — one key gives you access to
every available model.

Jan supports the Gonka Broker API, letting you run large open-source models from the cloud when your
local hardware is not enough, while keeping the same workflow you already use in Jan.

## Integrate Gonka Broker with Jan

<Steps>

### Step 1: Get Your API Key
1. Visit the [Gonka Broker Dashboard](https://app.gonkabroker.com/) and sign in
2. Create & copy a new API key (keys are prefixed `gnk-prx-`)

<Callout type='info'>
Ensure your account has sufficient balance. Your balance is shared across all available models.
</Callout>

### Step 2: Configure Jan

1. Navigate to the **Settings** page (<Settings width={16} height={16} style={{display:"inline"}}/>)
2. Under **Model Providers**, select **Gonka Broker**
3. Insert your **API Key**

### Step 3: Start Using Gonka Broker Models

1. Pick any existing **Chat** or create a new one
2. Select any model from the **model selector** under Gonka Broker
3. Start chatting
</Steps>

## Available Models Through Gonka Broker

Jan automatically fetches the models available on your Gonka Broker account. For custom configurations:

**Model Field Settings:**
- Leave empty to use the models fetched from your account
- Specify a model using its exact ID
- Available options can be found at [Gonka Broker Models](https://gonkabroker.com/models)

**Examples of Model IDs:**
- MiniMax M2.7: `MiniMaxAI/MiniMax-M2.7`
- Kimi K2.6: `moonshotai/Kimi-K2.6`

## Troubleshooting

Common issues and solutions:

**1. API Key Issues**
- Verify your API key is correct and not expired
- Check if you have sufficient balance in your Gonka Broker account
- Ensure you have access to the model you're trying to use

**2. Connection Problems**
- Check your internet connection
- Look for error messages in [Jan's logs](/docs/desktop/troubleshooting#how-to-get-error-logs)

**3. Model Unavailable**
- Confirm the model is currently available on Gonka Broker
- Check if you're using the correct model ID
- Verify the model provider is currently operational
